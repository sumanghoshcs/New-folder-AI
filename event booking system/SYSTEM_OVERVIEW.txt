✨ WHAT'S IMPLEMENTED

API Endpoints (13 total)
├── Authentication (2)
│   ├── POST /api/auth/register      - Register new user
│   └── POST /api/auth/login         - Login & get JWT token
│
├── Organizer Endpoints (5)
│   ├── POST   /api/events           - Create event
│   ├── GET    /api/events/organizer/list  - List organizer's events
│   ├── GET    /api/events/:id       - Get event details
│   ├── PUT    /api/events/:id       - Update event
│   └── DELETE /api/events/:id       - Delete event
│
├── Customer Endpoints (3)
│   ├── GET    /api/events           - Browse published events
│   ├── POST   /api/bookings         - Book tickets
│   └── GET    /api/bookings         - View bookings
│
├── Admin/Status (3)
│   ├── GET    /api/jobs             - List background jobs
│   ├── GET    /api/jobs/:id         - Get job details
│   └── GET    /api/health           - Health check

Database Tables (4)
├── users       - Store organizers and customers
├── events      - Event information with organizer relationship
├── bookings    - Customer bookings for events
└── jobs        - Background task queue with status tracking

Background Tasks (2)
├── Task 1: booking_confirmation
│   ├── Triggered: When customer books tickets
│   ├── Action: Log formatted email confirmation
│   └── Console: Shows booking details
│
└── Task 2: event_update_notification
    ├── Triggered: When organizer updates event
    ├── Action: Notify all customers with bookings
    └── Console: Lists all customers to notify

❓ COMMON QUESTIONS

Q: Can I run this multiple times?
A: Yes! The demo handles existing users gracefully and re-runs successfully.

Q: What if the port 3000 is already in use?
A: Edit .env file and change PORT=3000 to PORT=3001, then restart.

Q: Can I modify the code for the video?
A: Yes! Feel free to customize event names, user details, etc.

Q: Is the database persistent?
A: Yes. eventbooking.db is created once and persists between runs.

Q: How do I reset the database?
A: Delete eventbooking.db and restart the server (will auto-create).

Q: Can I use this for production?
A: It's production-ready for learning! For production, use PostgreSQL and Bull queue.
